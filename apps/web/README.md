# Web App (`apps/web`)

Next.js 16 frontend for the current athlete-first product surface.

## Stack

- Next.js 16 App Router
- React 19
- Tailwind CSS v4
- `next-themes`
- Supabase auth/data clients
- Vercel AI SDK 6 (`@ai-sdk/react`)
- Serwist service worker / PWA manifest

## Local development

From repo root:

```bash
pnpm --filter web dev
```

Default port: `3100`.

## Required env vars

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `NEXT_PUBLIC_API_URL`
- `NEXT_PUBLIC_WEB_URL`

Important implementation note:
- In `apps/web/src`, read public env values with static access like `process.env.NEXT_PUBLIC_SUPABASE_URL`.
- Do not use dynamic access like `process.env[name]` in browser/shared modules. Next standalone may not inline those values into client bundles, which can break production even when Azure app settings are correct.

Optional feature flags:

- `NEXT_PUBLIC_ENABLE_GOOGLE_AUTH`
- `NEXT_PUBLIC_ENABLE_DEMO`

## Current routes and behavior

- `basePath` is `/workout` (see `next.config.ts`).
- `/health` is the deployment health endpoint and resolves to `/workout/health` in production because of the base path.
- `/` is the current marketing/landing page.
- `/login` handles Supabase auth entry.
- `/dashboard/*` holds the shipped app surface.
- `/dashboard/body-map-3d` still exists but remains experimental and uses sample fatigue data.
- AI Coach streams through `/api/ai/stream`.
- Planned workouts use `/api/planned-workouts`.
- Integrations settings use `/api/integrations/status` plus provider action routes.
- Service worker output is generated by Serwist and disabled in development.
- In deployed environments, `NEXT_PUBLIC_API_URL` must point at the real API origin. Web CSP `connect-src` is built from that value at build time.
- Azure App Service should run this app on Node `24.x`, keep `alwaysOn` enabled, and probe `/workout/health` for Health Check.
- The GitHub deploy workflow smoke-tests `/workout/` after rollout, so production-only env/config issues should fail the deploy instead of silently shipping.

## Build

```bash
pnpm --filter web build

```
