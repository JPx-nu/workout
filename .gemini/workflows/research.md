# Research Workflow

1. Start with local docs and the nearest `AGENTS.md`.
2. Inspect the actual implementation files before proposing changes.
3. Use external research only for unstable or security-sensitive topics, or when the user asks for verification.
4. Prefer official framework, provider, and platform docs.
5. Fold the findings back into code, docs, and agent guidance so the repo stays internally consistent.
